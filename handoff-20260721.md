## 二、做了哪些关键决策

1. **逻辑删除而非物理删除** — 使用 `deleted_at` 时间戳，管理员和顾问可分类查看已删除项目
2. **PDF 导出用 pdfkit** — 安装在 `apps/api`，包含公司 Logo（通过 `COMPANY_LOGO_URL` 环境变量配置）
3. **用户端删除后立即软删除** — 调用 `DELETE /api/projects/:id` → `softDeleteProject()`，不真删数据
4. **管理端导出菜单** — 从项目详情抽屉中提供「导出 Excel」「导出 PDF」「导出 Markdown」三个按钮
5. **报价单暂不在本期实现** — 用户确认先不做独立报价单页面，导出功能已覆盖报表需求

#### web 端 TS 警告详情

```
apps/web/src/App.tsx:63
- identity 被推断为 string，应为 "investor"|"therapist"|"partner"|"other"
- stage 被推断为 string，应为 "exploring"|"site_selected"|"opening_soon"
```
原因：`useState` 的 `identity` 和 `stage` 初始值来自 `user.identity`（string 类型），导致状态类型宽泛化。
**临时方案**：不改也不影响运行，下拉框已限定了可选值。修复只需加 `as` 类型断言。

#### admin 端 TS 警告详情

```
apps/admin/src/App.tsx:70
- roleColor[v] / roleLabel[v] 报 TS7053，v 是 Ant Design Table render 的 any 类型
```
**临时方案**：不改也不影响运行，`role` 实际只有 'admin'/'consultant'。修复只需加类型断言。
