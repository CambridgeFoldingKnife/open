# 开馆助手 PRD 需求文档 1.0

---

## 一、用户端（小程序 / Web）

### 1.1 认证模块

用户通过邮箱注册登录，支持推荐码关联顾问。

| 功能点 | 描述 |
|--------|------|
| 注册 | 填写姓名、城市、手机号、密码、身份（投资人/康复师/合伙人/其他）、创业阶段；验证邮箱后完成注册。URL 参数 `?ref=推荐码` 自动关联顾问 |
| 登录 | 支持密码登录和邮箱验证码登录 |
| 获取当前用户 | 返回登录用户的基本信息和关联关系 |

**接口：**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/auth/register-email` | 邮箱注册（body 含 ref 参数，自动写入 referred_by_consultant_id） |
| POST | `/auth/login-email-password` | 邮箱+密码登录 |
| POST | `/auth/login-email-code` | 邮箱+验证码登录 |
| POST | `/auth/send-email-code` | 发送邮箱验证码 |
| GET  | `/auth/me` | 获取当前用户信息 |

---

### 1.2 项目管理模块

用户创建开馆项目，完成能力评估和场地信息填写后提交，生成门店定位与设备报价报告。

| 功能点 | 描述 |
|--------|------|
| 新建项目 | 填写创始人角色、城市、面积、预算、受众、服务、开业日期、团队等信息 |
| 填写评估 | 完成准备度评估（5 维）和能力评估（10 维），生成四象限定位和运营原型推荐 |
| 提交项目 | 确认提交后数据落库，顾问端可查看 |
| 查看项目列表 | 展示所有未删除的项目卡片（城市、面积、定位类型、能力分数） |
| 查看项目报告 | 四象限图、经营原型、设备报价、装修建议、盈亏测算、甘特图等完整报告 |
| 逻辑删除项目 | 用户删除项目，设置 deleted_at 标记；用户端列表中消失，顾问和管理员可查看已删除项目 |

**接口：**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST   | `/projects/draft` | 创建项目草稿 |
| POST   | `/projects/:id/submit` | 确认提交项目（触发评估计算） |
| GET    | `/projects` | 获取用户项目列表（自动过滤已删除） |
| GET    | `/projects/:id` | 获取单个项目详情 |
| DELETE | `/projects/:id` | 逻辑删除项目（设置 deleted_at） |
| GET    | `/bootstrap` | 获取当前用户信息 + 项目列表 |

---

## 三、顾问端 / 销售端（Web 后台）

顾问端复用管理端的页面框架，通过权限控制菜单和数据范围。

### 3.1 客户管理模块

| 功能点 | 描述 |
|--------|------|
| 查看客户项目 | 查看通过自己推荐码注册的客户的项目列表，支持搜索 |
| 项目详情 | 查看客户项目的完整报告（四象限、原型、报价、盈亏测算） |
| 已删除项目 | 查看客户已删除的项目（标记状态） |
| 项目阶段 | 标注当前阶段：待联系 / 已联系 / 已完成 |

**接口：**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/projects?query=deleted` | 查看自己客户的已删除项目 |
| GET | `/projects/:id` | 项目详情（自动限权为仅自己客户） |
| GET | `/users/:id/projects` | 某个用户的项目列表（仅限自己客户） |

---

### 3.2 报价单管理模块

| 功能点 | 描述 |
|--------|------|
| 查看报价单 | 查看客户项目的设备报价单（三档配置：基础版/标准版/专业版） |
| 修改报价单 | 修改报价单中的产品名、单价、数量等字段 |
| 删除报价项 | 删除某个报价条目 |
| 新增报价项 | 新增报价行（借用产品表中的产品，也可手动输入） |

> **重要**：顾问端的报价单修改仅在前端操作，**不落库**。产品数据统一由管理员在管理端维护。

**接口：** 无需额外后端接口，复用项目查询接口即可。

---

### 3.3 导出模块

| 功能点 | 描述 |
|--------|------|
| 报价单导出 | 导出 PDF / Markdown / Excel 格式，可供管理员导入沉淀到产品库 |
| 项目报告导出 | 导出客户项目的完整报告 |

**接口：**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/projects/:id/export?format=excel\|pdf\|md` | 三格式导出 |
| GET | `/projects/:id/quotation/export?format=excel\|pdf\|md` | 报价单单独导出（可选，后续实现） |

## 五、数据模型（简述）

| 表 | 用途 | 关键字段 |
|----|------|----------|
| `user_accounts` | 注册用户 | name, phone, email, city, identity, stage, referred_by_consultant_id, payload(JSONB) |
| `staff_accounts` | 员工/顾问 | name, email, role(admin\|consultant), referral_code(unique), active, title, phone |
| `projects` | 开馆项目 | customerId, consultantId, status, deleted_at, quadrant, capability/readiness JSON |
| `leads` | 销售线索 | projectId, assignedConsultantId, status, quoteAmount, expectedAmount |
| `catalog` | 产品目录 | name, category, unitPrice, unit, description |
| `follow_ups` | 跟进记录 | leadId, content, createdAt |
